ARCHITECTURE SUMMARY
================================================================================

Core architecture now uses one unified camera screen:
1. ActiveCameraScreen.tsx

Removed duplicate screens:
1. NoIntentScreen.tsx (removed)
2. IntentScreen.tsx (removed)

Unified behavior is selected via route param:
1. mode: 'wandering' | 'destination'

Global pipeline switch:
1. ENABLE_INTENT_MODE (src/config/modelConfig.ts)

Behavior matrix:
1. mode='wandering' -> lightweight ConvLSTM path
2. mode='destination' + ENABLE_INTENT_MODE=false -> lightweight ConvLSTM path
3. mode='destination' + ENABLE_INTENT_MODE=true -> intent-aware ConvLSTM path

2. Provide centralized timer/listener cleanup with ownership guards to reduce cross-screen race conditions.
3. Support skipSpeech flow, pre-TTS earcon playback, and listening readiness haptics.
4. Emit standardized audio lifecycle logs with [AUDIO-DEBUG] prefix.

Core state returned by hook:

2. isSpeaking

Core control methods:

3. transitionToListening

PREPROCESSOR (src/services/preprocessor.ts)
================================================================================

FrameData:
1. data: Uint8Array (RGBA)
2. width: number
3. height: number
4. timestamp: number
5. sequenceId?: number

ProcessedTensor:
1. data: Float32Array
2. shape: [batch, seq, channels, height, width]
3. processingTimeMs: number

FrameBuffer helpers:
1. addFrame
2. isReady
3. canPredictEarly
4. getFrames
5. getFramesPadded
6. getFramesBootstrapDoubled

Tensor/channel semantics:
1. Dynamic shape by active path:
   - [1, 20, 3, 128, 128] for no-intent path
   - [1, 20, 6, 128, 128] for intent-aware path
2. 3-channel packing math (tight RGB):
   - frameStride = height * width * 3
   - pixel base index = frameOffset + (y * width + x) * 3
3. 6-channel packing math (RGB + intent-safe padding):
   - frameStride = height * width * 6
   - RGB packed at packedBase + 0..2
   - slots packedBase + 3..5 reserved so addIntent writes are in-bounds
4. addIntent behavior remains unchanged and only writes intent-channel slots when enabled.

================================================================================
INFERENCE SERVICES
================================================================================

ConvLSTM without intent (convlstmWithoutIntentInference.ts):
1. Current baseline classification path
2. Uses 3-channel preprocessor allocation in ActiveCamera when intent path is disabled
3. Exposes initializeModel, runPrediction, cleanupModel

ConvLSTM with intent (convlstmWithIntentInference.ts):
1. Parallel service API with same interface
2. Selected by ActiveCamera when mode='destination' and ENABLE_INTENT_MODE=true
3. Uses 6-channel preprocessor allocation in ActiveCamera

YOLO (yoloInference.ts):
1. Exposes initializeYOLOModel, runYOLODetection, cleanupYOLOModel
2. Produces Detection[] with normalized bounding boxes
